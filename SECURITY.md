# Security policy

## Reporting a vulnerability

If you believe you’ve found a security vulnerability, please **do not** open a public GitHub issue.

Instead:

- Create a private report via GitHub Security Advisories (preferred), or
- Contact the maintainers with a clear description, impact, and reproduction steps.

## What to include

- Affected component(s) and configuration
- Steps to reproduce (proof-of-concept if possible)
- Expected impact and severity assessment
- Any suggested fix/mitigation

## Supported versions

This repository is currently pre-release. Security fixes are applied to `main` as needed.
