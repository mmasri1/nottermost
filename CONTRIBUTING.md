# Contributing

Thanks for contributing to **Nottermost**.

## Scope and principles

- Keep changes aligned with the repo goals in `README.md` (distributed-systems learning + cost/reliability trade-offs).
- Prefer **documenting decisions** over adding complexity.
- Keep the repo **mergeable**: small PRs, clear descriptions, and reversible changes.

## How to contribute

- **Bugs**: open an issue with repro steps, expected vs actual behavior, and any logs/metrics snapshots.
- **Features**: propose the problem statement first; include trade-offs (cost/latency/durability/ops).
- **Docs/diagrams**: welcome—especially around data models, scaling, and cost estimation.

## Pull requests

- Use the PR template.
- Include a **test plan** (even if manual).
- Call out **operational impact** (deploy/rollback, alerting changes, cost).

## Security

Please do not report vulnerabilities via public issues. See `SECURITY.md`.
